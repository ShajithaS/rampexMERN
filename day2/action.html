Here is the List of git commands

  1. git -v
    • Displays the installed Git version.
    • Helps verify whether Git is installed on your system.
  2. git init
    • Initializes a new Git repository in the current directory.
    • Creates a .git folder, which stores all version control data.
    • Converts an existing project into a Git-tracked repository.
  3. git status
    • Shows untracked, staged, and modified files.
    • Helps track file changes before committing.
  4. git add
    • Moves files from the working directory to the staging area.
    • git add . stages all modified files.
    • Prepares files for the next commit.
  5. git config
    • git config --global user.name "Your Name"
    • git config --global user.email "your.email@example.com"
    • Configures Git username and email (required for commits).
    • --global applies the settings system-wide.
    • --local applies settings only for the current repository.
  6. git remote add origin
    • git remote add origin https://github.com/user/repo.git
    • Links the local Git repository to a remote repository.
    • origin is the default name for the remote.
    • Allows pushing/pulling code from GitHub or other Git servers.
  7. git push
    • git push origin main (or) git push -u origin main
    • Pushes committed changes to the remote repository.
    • -u sets the upstream branch for future git push commands.
    • Requires authentication (if pushing to a private repo).
  8. git branch (Manage Branches)
    • git branch (or) git branch
    • Lists all existing branches in the repository.
    • Creates a new branch (git branch feature-xyz).
    • The main branch is usually named main or master.
  9. git checkout (Switch Branches)
    • git checkout (or) git checkout -b
    • Switches to another branch.
    • -b creates and switches to a new branch in one step.
  10. git log
    • View Commit History
    • Shows a list of past commits with details like:Commit ID,Author,Date,Commit message
  11. git pull
    • git pull origin main
    • Fetches the latest changes from the remote repository.
    • Merges remote updates into the local branch.
    • Avoids conflicts when multiple people work on the same project.
  12. git stash
    • Temporarily saves uncommitted changes and resets the working directory.
    • Useful when switching branches without committing changes.
  13. git stash apply
    • Restores the last stashed changes without deleting them.
    • If multiple stashes exist, use git stash list and apply a specific stash: example - git stash apply stash@{1}
